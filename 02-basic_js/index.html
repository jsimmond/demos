En en esta demo, mostrare como podemos usar JavaScript para modificar el contenido de una pagina, como también para validar los datos de un formulario antes de que los datos sean enviados al servidor. JavaScript es un lenguaje orientado a objetos basado en prototipos, lo que significa que en vez de crear objeto usando un constructor, como lo hacen usualmente en Java, pueden modificar el prototipo de un objeto para crear objetos nuevos.


Como funciona esto? Cuando se carga un archivo .html en un navegador, queda disponible un objeto llamado document. Podemos usar funciones como getElementById() y querySelector() para encontrar los elementos que queremos modificar (como una tabla, un titulo, un formulario, etc.) y después modificar sus valores para actualizar la pagina. Esta pagina es una referencia completa a los métodos y atributos del objeto document.


La ventaja de usar JavaScript es que podemos alterar el contenido de una pagina sin navegar hacia otra pagina.


Ejemplos:

  1. En este ejemplo pueden ver como use getElementById() para agregar texto a un párrafo cuando el usuario presiona un botón. Ojo que en este ejemplo, he incluido el código JavaScript en la misma pagina. Usualmente guardamos el código JavaScript en archivos separados, así podemos reusar las funciones que definimos en un archivo en varias paginas.
  2. En el segundo ejemplo pueden ver como use una caja de texto para actualizar una lista que se encuentra en la pagina. De nuevo use el "onclick" de un botón para gatillar la función JavaScript, pero ahora creo nuevos elementos y los agrego a document para poder actualizar la pagina. Ah, y use querySelector() en vez de getElementById().
  3. Finalmente, he incluido un ejemplo de un formulario con validación. Hay muchas formas distintas de validar un formulario, en este caso escribí una función que se ejecuta cuando se presiona el botón para enviar los datos. Ojo que para ejecutar este ejemplo, deben preocuparse que el archivo ./cgi-bin/process_data.py tenga permiso de ejecución, y que hayan levantado un servidor CGIHTTPServer en el directorio 02-basic_js. Este ejemplo levanta una alerta, en la practica NO se recomienda hacer esto. En la próxima demo veremos como insertar los mensajes de error en el mismo formulario, lo que es mucho mejor.
